## ADDED Requirements

### Requirement: El README debe contener descripción del proyecto
El README SHALL incluir una descripción clara del propósito del proyecto, identificándolo como un scraper de propiedades basado en Playwright.

#### Scenario: Descripción presente
- **WHEN** un usuario abre el archivo README.md
- **THEN** puede leer una descripción que explica que es un scraper de propiedades

### Requirement: El README debe contener instrucciones de instalación
El README SHALL incluir instrucciones claras para instalar las dependencias del proyecto.

#### Scenario: Instrucciones de instalación visibles
- **WHEN** un usuario nuevo quiere usar el proyecto
- **THEN** encuentra instrucciones para instalar con pip install -e .

### Requirement: El README debe contener examples de uso
El README SHALL incluir examples showing how to use the scraper.

#### Scenario: Usage examples presentes
- **WHEN** un desarrollador quiere usar el scraper
- **THEN** encuentra examples de código para ejecutar el main.py

### Requirement: El README debe documentar la estructura del proyecto
El README SHALL incluir una sección que muestra la estructura de directorios y archivos.

#### Scenario: Estructura documentada
- **WHEN** un colaborador quiere entender el código
- **THEN** puede ver la estructura del proyecto en el README
